# 배포 및 유지보수 가이드 (Deployment & Maintenance Guide)

웹사이트를 배포한 이후, 수정 사항이 생겼을 때 대처하는 방법입니다.

## 1. 데이터(Content) 수정

매물 정보, 파트너사 목록, 고객 문의 확인 등 **데이터**와 관련된 변경은 코드를 수정할 필요가 없습니다.

* **방법**: [Supabase 대시보드](https://supabase.com/dashboard) 접속
* **Table Editor** 메뉴로 이동하여 엑셀처럼 데이터를 수정/추가/삭제하면 **실시간으로 웹사이트에 반영**됩니다.
* **Tip**: 관리자 페이지(`http://your-site.com/admin`)를 구축해 두었으므로, 이곳에서 로그인하여 수정하는 것이 가장 편리합니다.

## 2. 디자인 및 기능(Code) 수정

화면 레이아웃, 색상, 새로운 기능 추가 등 **코드** 자체가 바뀌어야 하는 경우입니다.

### 2-1. 수정 절차

1. **로컬 개발**: VS Code를 열고 코드를 수정합니다.
2. **테스트**: `npm run dev` 명령어로 로컬에서 잘 작동하는지 확인합니다.
3. **빌드(Build)**: 배포용 파일을 생성합니다.

    ```bash
    npm run build
    ```

    * 이 명령을 실행하면 `dist` 폴더가 생성(또는 갱신)됩니다.

### 2-2. 배포 (Deployment)

수정된 `dist` 폴더를 서버에 올리는 과정입니다.

* **Vercel / Netlify 사용 시 (추천)**:
  * GitHub에 코드를 푸시(`git push`)하면, **자동으로 감지하여 재배포**됩니다. (가장 편한 방법)
* **일반 호스팅 (FTP) 사용 시**:
  * FTP 프로그램(FileZilla 등)을 이용해 `dist` 폴더 안의 모든 파일을 호스팅 서버의 `public_html` 폴더로 덮어쓰기 합니다.

## 3. 긴급 복구

만약 수정하다가 사이트가 깨졌다면?

* **이미지/데이터 문제**: Supabase나 로컬 파일 백업본을 확인하세요.
* **코드 실행 오류**: 터미널의 에러 메시지를 확인하거나, Git을 사용 중이라면 이전 버전으로 되돌리기(`git revert`)를 하세요.

## 4. 문의

해결되지 않는 기술적 문제는 개발자(또는 유지보수 담당자)에게 다음 정보를 전달하세요.

1. 어떤 화면에서 문제가 발생했는지 (스크린샷)
2. 어떤 동작을 했을 때 발생했는지
3. 에러 메시지가 뜬다면 그 내용
